STA tools

This set of tools allows you to perform operations on STA data. Using these tools sends a new request to the STA server. The tools translate the information entered in the dialog boxes and complete the request URL with this data in order to send a new request to the API.

With these tools you can select, filter data, expand related entities from the current entity, sort data, and change the number of records returned, among other things.

Select Columns

Setting Up the Node:

How to Use the Node:

The purpose of the Select Columns tool is to generate a new table containing only the columns selected in the dialog displayed. The selection will be internally applied as a new request to the STA service.

This example contains the original data with all columns:

To display the dialog, double right-click on the node. Below you can see the dialog displayed, showing all the columns from your data. Deselect the columns that you do not want to include in the new table, leaving selected only those you want to keep. After making the selection, press OK button.

This is the result, a new table containing only the selected columns.

Now, the attributes of your data contain only the selected columns. To restore the original ones, you must continue from the previous node.

Take a look at the new request that TAPIS internally makes:

https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Observations?$select=@iot.selfLink, id, phenomenonTime, resultTime, result, resultQuality, validTime, parameters

Select Row

Setting Up the Node:

How to Use the Node:

The purpose of the Select Row tool is to generate a new table containing only the row selected (a resgister) in the dialog displayed.

This example contains all the original rows:

To display the dialog, double right-click on the node. Below you can see the dialog displayed, showing all the rows from your data. Select the register you want to keep (using the radio button on the left) and press the OK button, located at the bottom of the dialog.

After the new request to the STA service (remember that this tool creates a new request with the required query), you will obtain a table containing only the selected row.

Take a look at the new request that TAPIS internally makes:

OGC query URL: https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Datastreams(11)

You can obtain same result using Select Resource
